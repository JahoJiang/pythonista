1. 文件权限与目录配置

1.1 用户与用户组

Linux 是一个多人多任务系统，允许多个用户登录系统。

一个用户 User 可以隶属于某一个或多个群组 Group，当一个用户创建了一个文件时，他就是这个文件的“文件拥有者”。

root用户拥有最高权限，可以访问任何用户创建的文件。

1.2 文件与目录权限

在 Linux 中，每一个文件都具有用户 User、所属群组 Group 及 其它人 Others 三种身份所对应的权限。

CMD：ls 命令
  • -a：-all 列出目录下的所有文件，包括以 . 开头的隐含文件
  • -l：除了文件名之外，还将文件的权限、所有者、文件大小等信息详细列出来。
  • -s：–size 以块大小为单位列出所有文件的大小
➜  dir ls -als
total 8
0 drwxr-xr-x   4 jaho  staff  128 Dec  1 22:52 .
0 drwx------@ 10 jaho  staff  320 Dec  1 22:47 ..
8 -rw-r--r--   1 jaho  staff   12 Dec  1 22:52 file
0 drwxr-xr-x   2 jaho  staff   64 Dec  1 22:47 subdir

第2行：为文件总的大小。

第3-6行：

  1. 第一列为块大小
  2. 第二列为文件权限，开头第一个字符代表文件类型，d 代表目录，- 代表文件，l代表链接文件。接下来的字符三个一组，一次代表 User、Group 和 Other 的权限，其中 r 代表可读，w 代表可写权限，x 代表可执行，- 没有权限。
CMD：修改文件权限命令
  • chgrp：修改文件所属群组
  • chown：修改文件拥有者
  • chmod：修改文件权限
目录与文件权限意义

对于文件：

  • r：可读
  • w：可编辑、新增、修改文件内容（但不能删除
  • x：可被系统执行

对于目录：

  • r：读取目录结构列表
  • w：可建立新的文件和目录、删除或更名已经存在的文件和目录（不管文件权限如何）以及移动该目录内的文件、目录位置。
  • x：能否进入该目录

1.3 目录配置

FHS：Linux 的文件系统分层标准。

一些概念：

  1. 不变目录代表其中的数据不会经常变动，例如函数库，主机配置文件等。

  2. 可变目录代表其中的数据经常修改，例如日志文件。

  3. 可分享 代表可分享给其他系统挂载使用的目录
  4. 不可分享 代表仅与自身机器相关的文件等

如 /etc 为存放配置文件，属于不变、不可分享目录。

实际上，按照用途，分为：

  1. /root，根目录，与启动系统有关
  2. /usr，与软件安装/执行有关
  3. /var，与系统运行过程有关
目录树

在 Linux 中，所有文件与目录都是由根目录开始的，然后一个个的分支下来，因此称为目录树：

  • 目录树的根节点为根目录 （/，root）
  • 每一个目录不止可以使用本地的文件系统，还可以使用网络上的文件系统，如利用 NFS 服务器挂载特定目录。
  • 每一个文件在此目录树中的文件名（包含完整路径）都是独一无二的。

2. 文件与目录管理

CMD：目录相关命令
  1. cd：切换目录
  2. pwd：显示当前目录（完整路径
  3. mkdir：创建一个新目录
  4. rmdir：删除一个空目录

2.1 关于执行文件路径的变量：$PATH

为什么能够在任何地方执行 ls ? 因为系统会根据环境变量 PATH 的设置去每个 PATH 定义目录下查找文件名为 ls 的可执行文件，如果 PATH 定义的目录中存在多个同名文件，那么率先被找到的先被执行。

可以执行 echo $PATH 打印 PATH 变量。

➜  dir echo $PATH
/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin

这个变量内容由一堆目录组成，每个目录用:隔开，有顺序之分。

CMD：文件复制，移动和删除

  1. cp [源文件] [目标文件]，文件复制，复制后的目标文件权限与源文件权限可能不同 。

  2. mv [-fiu] [源地址] [目标地址]，文件移动，也可用于重命名

    • -f：强制，如果目标已存在则覆盖

    • -i：若存在，则询问是否覆盖

    • -u：若存在，且源文件更新，才会选择覆盖

  3. rm [-fir] [文件名]，文件删除

    • -f：强制，忽略不存在的文件，不会出现警告信息

    • -i：交互模式，删除前进行询问

    • -r：递归模式，常用于目录删除，比较危险

2.2 文件查看

CMD：直接查看文件内容
  • cat [文件名]，将一个文件内容连续的打印到屏幕
    • -n：显示行号，空白行也会有行号
    • -b：仅对非空白行计算并显示行号
  • tac [文件名]，cat命令的反转模式，从最后一行开始显示到第一行。
  • nl [-bnw] [文件名]，添加行号打印，行号打印的样式更多
CMD：其它
  • more [文件名]，翻页显示内容，光标位于最后一行，显示当前显示的百分比。
    • space，向下翻页
    • enter，向下一行
    • /字符串，在当前显示内容中，向下查找
    • :f，显示出文件名以及目前显示的行数
    • q，离开more
    • b：往回翻页
  • less [文件名]，比more更具弹性，可使用 pageup，pagedown 等功能随时向上向下查看与查找
  • head [-n number] [文件名]，显示文件中前几行，默认前10行
  • tail [-n number] [文件名]，显示文件中后几行，默认后10行
  • 利用管道符（前面命令的输出是后面命令的输入），可以取例如后20行的前10行：tail -n 20 file | head -n 10
  • od [-t Type] [文件名]，用来查看非纯文本文件
  • touch [文件名] 可以修改文件的文件时间或者创建新文件

2.3 文件查找

CMD：文件查找
  • which [文件名]，根据 PATH 查找执行文件的文件名
  • whereis [文件名]，在一些特定目录中查找
  • locate [文件名]，根据一个文件索引数据库进行查找，如果数据库没有更新就找不到，需要执行 updatedb 命令
  • find [文件名]，直接查找硬盘，速度很慢

3. 文件系统部分知识

3.1 硬链接 Link 与符号连接 Symbol Link

硬链接

硬链接会在目录树上创建新的 inode，产生新的文件名，但不创建新的文件。由于Linux 中，文件名只与目录有关，而文件内容与 inode 有关，读取文件内容时，需要指向正确的inode 号码，因此，就可以多个文件名对应同一个 inode 号码，这就是硬链接。

这样，删除了一个文件名，还可以使用其它文件名打开文件。

符号连接

符号连接就是一个独立的文件，这个文件会让数据的读取指向它连接的那个文件的文件名。当源文件被删除，符号链接指向的文件会打不开。类似于 Windows 的快捷方式。

CMD：硬链接与符号连接
  • ln [源文件名] [文件名]，创建硬链接
  • ln -s [源文件名] [文件名]，创建符号连接

3.2 内存交换分区 Swap 创建

  1. gdisk 命令划出一块区域
  2. mkswap [设备文件名] 将该区域格式化为交换分区格式
  3. swapon [设备文件名] 启动该 swap 设备
  4. 通过 free 与 swapon -s 来观察内存使用量

4. 计划任务 crontab 命令

Linux 上的计划任务分为两种，一种是例行性的，每隔一段时间执行一次，一种是突发性或者说一次性的，执行完一次就不再执行。例如 updatedb命令就是例行性的，一般一天一次。

CMD：at （仅执行一次

at 命令仅能被 /etc/at.allow 的用户或者不在 /etc/at.deny 中的用户执行

  • at TIME 进入 at 交互，再输入任务，最后 ctrl +d 退出。
CMD：crontab （循环执行

crontab 命令仅能被 /etc/cron.allow 的用户或者不在 /etc/cron.deny 中的用户执行，前者优先级更高。

当用户使用 crontab 命令建立计划任务，该项任务就会被记录到 /var/spool/cron/ 中，而且是以账户来作为判断依据。例如，用户 user_a 创建的任务就会被记录到 /var/spool/cron/user_a 中。

原理

cron由crond守护进程和一组表（crontab文件）组成。 crond守护进程是在系统启动时由init进程启动的，受init进程的监视，如果它不存在了，会被init进程重新启动。 这个守护进程每分钟唤醒一次，并通过检查crontab文件判断需要做什么。

  • crontabl [-u username] [-l|-e|-r]，其中
    • -u：只有 root 才能执行这个任务或者帮助其他用户建立/删除任务
    • -e：进入 vi 模式编辑 crontab 的任务内容
    • -l：查看crontab 的任务内容
    • -r：删除所有的crontab 的任务内容，仅删除部分，需要 -e 进入 vi 模式编辑。

每项 crontab 任务的格式：

# 分            时            日            月            周            需要执行的命令
    0-59    0-23    1-31        1-12    0-7

特殊字符

特殊字符 代表含义
* 星号 对应的时间单位为任何值都可，如任何一天的12:00种，0 12 * * *
, 逗号 代表分隔时间段，如 0 3,6 * * * 代表每天的 3:00 与 6:00。
- 减号 代表时间段，如0 3-6 * * * 代表对于小时，3，4，5，6都适用
/n 斜线 n代表数字，例如 /5 代表每隔五个时间单位。

5. 进程管理

在 Linux 中，程序一般放置在物理磁盘中，然后通过用户的执行来出发，触发后会加载到内存中成为一个个体，这就是进程。为了让操作系统可以管理这个进程，进程会根据执行者权限/属性等参数以及进程所需要的脚步或数据等，基于进程一个 PID，即进程ID。PPID 就是这个进程的父进程的 PID。

例如，我们远程登录到一台服务器，获取到 shell，例如 zsh（在 /bin/zsh 下），实际上就是系统启动了 zsh 程序，并为它分配了一个 PID。使用 ps -l 命令即可查看。

  • 程序：二进制程序，以物理形式放置在存储媒介中。
  • 进程：程序被触发后，操作系统将程序代码、所需数据和管理相关数据加载到内存中，并给予这个内存单元一个标识符 PID，可以认为，一个进程就是一个运行中的程序。

Linux 的程序调用通常称为 fork-and-exec 的流程：进程都会借由 fork 调用来产生一个一模一样的子进程，然后被复制出来的子进程（被称为中间临时进程）再借由 exec 调用来执行实际要执行的进程，最终才产生了我们想要的一个子进程（被称为最终子进程）。

一些常驻在内存中的进程被称为服务（详见6. 系统服务），例如上述的 crond 进程。

5.1 任务管理

CMD：任务管理相关命令

任务管理，用于当前 shell 中运行的任务，每个任务都是当前 shell 的子进程。

  • &，追加到命令末尾，则可将其将放置在后台运行，以继续执行其它命令，此时 shell 会给其一个任务号码
  • ctrl +z，将目前的任务暂停（进程挂起）并暂时丢到后台中。例如正在vim中，可使用它将 vim 放到后台运行再切换回来。
  • jobs [-lrs]，查看当前后台的任务状态：
    • -l，除了列出任务号与对应的命令还列出 PID
    • -r，仅列出正在后台运行的任务
    • -s，仅列出在后台暂停了的任务
  • fg %任务号码，将后台的任务切换到前台运行
  • bg %任务号码，将后台任务的状态切换为运行中
  • kill -signal %任务号码，删除或重新启动任务，选项有
    • -l，列出 signal 可选值
      • -1，重新读取一次参数的配置文件，类似 reload
      • -2，代表由键盘输入 ctrl + c 的操作，产生 SIGINT 中断信号，通知程序，程序可以处理这个信号，甚至选择忽略。
      • -9，立刻强制删除一个任务。kill -9相对强硬一点，会产生 SIGKILL 强制终止信号，这个信号是程序不能处理的，这个程序会被直接停止运行。
      • -15，以正常的方式终止一项任务。与 -9 不一样。因为kill -15 产生的是 SIGTERM 中止信号只是通知对应的进程要进行"安全、干净的退出"，程序接到信号之后，退出前一般会进行一些"准备工作"，如资源释放、临时文件清理等等，如果准备工作做完了，再进行程序的终止。甚至，程序可以选择忽略此信号。

ctrl +c 原理

前台进程使用 ctrl +c 关闭，而后台任务使用 kill 命令。执行 ctrl +c 时，会发送 SIGINT （中断）信号给前台进程组中的所有进程，强制终止程序的执行。SIGINT 与 SIGTERM 的区别主要在于应用程序，一般来说产生的效果一致，但是程序开发者可以根据自己的需要来对两种信号执行不同的处理程序。

CMD：后台运行

ctrl +z 的后台是 shell 的后台，一旦 shell 关闭，这些子进程都会终止运行。如果需要把任务放在系统后台运行，不因 shell 的关闭受到影响，可以使用 nohup 命令。

  • nohup [命令与参数]，在前台执行任务
  • nohup [命令与参数] &，在后台执行任务

5.2 进程管理

CMD：查看进程
  • ps [选项]，查看进程信息，选项有：
    • -A，显示所有进程
    • -a，不显示终端有关的进程
    • -x，通常与a一起使用，列出完整信息
    • -l：较长、较详细地将 PID 与相关信息列出来

例如，仅查看自己的 shell 相关的进程

➜  dir ps -l
  UID    PID        PPID    F         CPU    PRI NI SZ             RSS     WCHAN S ADDR    TTY            TIME         CMD
  501    57913    57911    4006    0        31    0  4334824  4484     -     S 0         ttys000 0:00.41 -zsh

其中：

  • UID/PID/PPID：被该UDI拥有/ 进程PID / 父进程 PID
  • F：说明这个进程的权限
  • C或CPU：代表 CPU 使用率
  • PRI/NI：PRI 为进程优先级，由内核管理，我们无法干预，NI 类似 PRI，我们可以干预
  • ADDR/SZ/WCHAN：都与内存使用有关
  • TTY：登录者的终端位置
  • TIME：使用 CPU 的时间
  • RSS：进程占用的固定的内存量 KB
  • COMMAND/CMD：该进程的实际命令

ps 命令显示的是一个时间点的进程状态，如果要动态的查看进程信息，则应该使用 top 命令

  • top [-d 数字] ｜ top [-bnp]，
    • -d，后面接秒数，即整个页面刷新的时间间隔
    • -b，以批量的形式执行 top
    • -n，与 -b 搭配，意义是需要执行几次 top 的输出结果
    • -p，指定某个 PID 进行查看
进程如何管理

后台进程根据接受到的信号进行管理，kill 所使用的信号：

代号 名称 内容
1 SIGHUP 启动被终止的进程，让PID重新读取配置文件，类似于重新启动
2 SIGINT 相当于用键盘输入 ctrl +c 来中断一个进程的运行
9 SIGKILL 代表强制中断一个进程的执行，如果该进程执行到一半，那么未完成部分可能会有半成品，例如 vim 会有 .filename.swap文件遗留
15 SIGTERM 通知程序以正常的方式终止，如果进程已经发生问题，无法处理信号，那么也无法终止它
19 SIGSTOP 相当于用键盘输入 ctrl +z 来暂停一个进程的运行
CMD：查看系统资源
  • free：查看内存使用情况
  • uname：查看系统与内核相关信息
  • netstat -[atunlp]，查看网络与socket信息，
    • -a，将目前系统中所有的连接、监听、socket信息都列出来
    • -t，列出 tcp 网络封包的信息
    • -u：列出 udp 网络封包的信息
    • -n：不以进程的服务名称而是以端口号来显示
    • -l：列出目前正在监听的服务
    • -p：列出该服务的PID
  • fuser：查看被使用的文件
  • lsof：列出被进程使用的文件名称

6. 系统服务 daemon

daemon 指的是守护进程，service 总是依靠 daemon 来实现的。

目前 Linux 普遍采用 systemd 来启动服务。

6.1 通过 systemctl 管理单一服务

CMD: systemctl
  • systemctl [command] [unit]，根据 command 来对 unit 服务执行操作，command有：
    • start：启动 unit
    • stop：关闭
    • restart：重启
    • reload：不关闭 unit 的情况下重新加载配置文件
    • enable：使其在系统启动的时候被启动（开机自启
    • disable：关闭开机自启
    • status：查看状态
    • is-active：是否运行中
    • is-enable：是否开机自启

我们可以编写一个shell脚本（.sh文件），然后设计一个 .service文件，将其放置在/etc/systemd/system/ 下面，注册为一个服务，再根据 systemctl 来管理。
